# Complete Overview: Structure of the Design System

The **Nuxt Design System** can be extended to include various **component states**, advanced **color variations**, and additional **utility classes**. Below is a structured summary that highlights how to organize colors, typography, spacing, borders, shadows, animations, and custom token-based classes ‚Äî all in **one cohesive design system**.

---

## 1. Color System

We define a **color system** subdivided into four main categories, complemented by status states and neutral colors:

- **Primary:** Main brand color.  
Examples: Buttons, links, and primary actions.
- **Secondary:** Supporting colors.  
Examples: Secondary buttons, informative sections, or complementary accents.
- **Tertiary:** Accent colors.  
Examples: Highlights, emphasis for certain UI elements.
- **Quaternary:** Additional supporting colors.  
Examples: Subtle backgrounds and secondary actions.
- **Neutral:** Grays and background colors.  
Examples: Text elements, disabled states, and general backgrounds.
- **States:** Status-dependent color variations for interactions.
- `-hover`: Hover state
- `-active`: Active state
- `-focus`: Focus state
- `-disabled`: Disabled state
- **Utility Colors:** Feedback or alert colors.  
Examples: `$success`, `$warning`, `$error`, `$info`

> **Tip:** These categories ensure clarity and consistency across complex interfaces.

---

## 2. Typography System

The typography system is defined by **font sizes**, **weights**, and **line heights**:

- **Font-Sizes:** `$small`, `$medium`, `$large`, `$xl`, etc.  
Examples: Small labels, headings, and body text.
- **Font-Weights:** `$light`, `$regular`, `$bold`  
Examples: Standard, bold, or lighter text uses.
- **Line-Heights:** `$tight`, `$normal`, `$loose`  
Controls readability and spacing in paragraphs or headings.
- **Letter-Spacing:** `$tight`, `$normal`, `$wide`

> **Remember:** Consistent typography is crucial for readability and brand identity.

---

## 3. Spacing

Standardized spacing for **margins**, **paddings**, and **grid gutters**:

- **Paddings/Margins:** `$spacing-1`, `$spacing-2`, ..., `$spacing-10`  
A scalable set of spacing tokens (e.g., `4px`, `8px`, `16px`, ...).
- **Grid-Gutters:** `$gutter-small`, `$gutter-medium`, `$gutter-large`  
Helps maintain consistent alignment and layout across columns.

---

## 4. Borders

Border tokens for **stroke widths** and **border radii**:

- **Border-Widths:** `$thin`, `$normal`, `$thick`  
Examples: Input fields, cards, and other containers.
- **Border-Radius:** `$none`, `$sm`, `$md`, `$lg`, `$full`  
Examples: Rounded buttons or subtly rounded corners on cards.

---

## 5. Shadows

Utilize shadows for **hover** and **active** states:

- **Box-Shadows:** `$shadow-small`, `$shadow-medium`, `$shadow-large`  
Examples: Modals, active buttons, and cards.

Shadows help give depth and emphasis to interactive UI elements.

---

## 6. Animations

Animations and transition settings for smooth user experiences:

- **Transition-Durations:** `$fast`, `$normal`, `$slow`  
Examples: Hover transitions and fade effects.
- **Transition-Timings:** `$ease`, `$ease-in`, `$ease-out`  
Examples: Animation curves for toggling components or modals.
- **Keyframes:** Reusable animations (e.g., loading spinners, fade-in effects).

---

## 7. Custom Classes

All tokens should use a **`$` syntax** for UnoCSS integration. Examples:

- **Colors:**
- `bg-$primary`, `bg-$primary-hover`, `text-$neutral-disabled`
- **Typography:**
- `font-$small`, `font-$medium-bold`
- **Spacing:**
- `p-$spacing-2`, `m-$spacing-4`
- **Borders:**
- `border-$thin`, `rounded-$lg`
- **Shadows:**
- `shadow-$medium`

> **Ready to dive in?** Start by defining your **colors** using Radix Colors as a base, then map them to your `$primary`, `$secondary`, `$tertiary`, `$quaternary`, `$neutral`, etc.
